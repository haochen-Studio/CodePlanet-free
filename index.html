📖 网安靶场 - 现实攻击防御教程

1. 攻击背景说明

本靶场模拟真实的校园服务器SSH暴力破解攻击（22端口），SSH是Linux服务器远程登录的常用端口，黑客会通过暴力破解尝试获取服务器权限。

2. 防御原理

我们需要使用iptables（Linux防火墙工具）关闭22端口，阻止黑客的攻击流量。
核心指令：iptables -A INPUT -p tcp --dport 22 -j DROP
指令说明：
- iptables：Linux防火墙命令
- -A INPUT：添加规则到输入链
- -p tcp：指定TCP协议
- --dport 22：目标端口为22
- -j DROP：丢弃符合条件的数据包

3. 操作步骤

① 在终端输入上述防御指令
② 点击回车执行指令
③ 观察服务器状态是否从"遭受攻击中"变为"安全"
④ 在10分钟内完成所有防御操作即可通关

4. 注意事项

本靶场为模拟环境，真实生产环境中关闭22端口前需确认是否有合法的远程登录需求，也可改为限制指定IP访问22端口。
